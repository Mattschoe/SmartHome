How to read this
The prototype is the source of truth for layout, spacing and interaction. This doc explains the intent behind it — the design system, the component contracts, the behaviors, and where to plug in real device data. Where the two disagree, the prototype wins on pixels, this doc wins on behavior.

02  Layout & grid

One screen, full-bleed. A soft sage surface (#B2C488) fills the viewport; three cream cards float on it in a fixed 3-column row. Nothing scrolls at the page level — only content inside a card scrolls.

LEFT · 288px fixed
Date/time header · 2×2 climate stats · Apps grid (scrolls)
CENTER · flex 1 (min 346px)
Room chips · brightness dial · warmth swatches · audio (speakers + volume)
RIGHT · flex 1.12 (min 392px)
Media / Calendar tab switch · scrolls · bottom fade cue
  • Outer frame: the prototype wraps the screen in a dark bezel (#1b1a16, radius 30, pad 14). In production this is the device itself — render the sage surface edge-to-edge, no bezel.
  • Surface padding: 24px top/bottom, 26px left/right. Card gap: 18px. Card radius: 22px (left card's inner blocks 15–18px).
  • Card style: background #FAF8EA, 1px border #A7BB7C. No drop shadow on cards — separation comes from the border + surface contrast.
  • Overflow rule: the whole screen is exactly one viewport tall. Left "Apps" block and right panel are the only scroll regions (overflow-y:auto). Right panel shows a 40px bottom gradient fade as a scroll affordance.

05  Signature interactions

Brightness dial (half-arc, drag)
  • SVG 260×160 viewBox. Semicircle from 180° (left, 0%) to 0° (right, 100%). Center of arc at (130,140), radius 116.
  • Track = full arc in #ECE6CF, 17px round cap. Value = arc from left up to the knob, stroked in the current warmth color.
  • Knob = white ⌀24 circle, 3.5px stroke in warmth color, sits at the arc end.
  • Center bulb glyph (Material lightbulb) with a warm radial glow whose radius + opacity scale with brightness. Tapping the bulb toggles the light on/off.
  • Drag math: pointer angle from center → degrees clamped 0–180 → value = round((1 − deg/180) × 100). Listen on window pointermove/up so drags survive leaving the SVG. Account for the SVG's rendered scale vs viewBox. Copy this math verbatim from the prototype.
  • Dragging also forces the light on. When off: arc greys to #cdc7ab, glyph → light_off, label reads "Off".
  • Label below: brightness % at 32px, or "Off". touch-action:none on the SVG.
Warmth swatches
  • Five ⌀34 circles: Candle → Warm → Soft → Neutral → Cool (fixed color temps, orange → pale blue).
  • Selected = scale 1.08 + double ring halo (2px card-colored gap, then 4.5px accent-of-warmth ring). Selecting also turns the light on.
  • The chosen warmth recolors the dial's value arc + knob — the two controls are visually linked.
Volume slider (drag)
  • 10px track #ECE6CF, fill #839958, ⌀22 white thumb with shadow. Flanked by volume_down icon and a right-aligned % readout (42px wide).
  • Drag math: (clientX − trackLeft) / trackWidth, clamp 0–1, ×100. Same window-listener pattern as the dial.
Room selection & per-room state
  • Everything in the center card is per-room. Brightness, on/off, warmth, volume, and audio-playing state are all keyed by room. Switching the room chip swaps the whole control state — no animation needed, just re-render.
  • Rooms: Living room, Kitchen, Bedroom, Bathroom. Selected chip = filled accent; idle = white with sage border.
  • Audio "join" affordance: if the selected room has no audio playing, show a dashed "Join the music in {source}" button instead of the now-playing line. Tapping it starts playback in that room (sets nowPlaying = true).
Media / Calendar tabs
  • Pill segmented control, one active at a time. Media: search, now-playing w/ scrubber, transport row (shuffle/prev/play/next/repeat), vertical "Up next" queue, horizontal-scroll "Playlists" rail (scroll-snap).
  • Calendar: month grid (today = accent cell), "Today" agenda list with colored dots, "To-do" checklist. Prev/next month arrows.
  • Panel scrolls vertically; 40px bottom fade signals more content.

06  State shape

Model as a single store keyed by room. This mirrors the prototype's state exactly — port it directly.

type Warmth = 'Candle'|'Warm'|'Soft'|'Neutral'|'Cool'; type RoomId = 'Living room'|'Kitchen'|'Bedroom'|'Bathroom'; interface RoomState { brightness: number; // 0–100 on: boolean; warmth: Warmth; volume: number; // 0–100 audioPlaying: boolean; } interface DashboardState { activeRoom: RoomId; rooms: Record<RoomId, RoomState>; speaker: RoomId | 'Whole home'; audioSource: RoomId; // where music originates panel: 'media' | 'calendar'; accent: 'Forest'|'Sage'|'Teal'|'Rose'; clock24: boolean; userName: string; }
  • Clock ticks every 20s (setInterval) — good enough for a minute display.
  • Persist accent, clock24, and last room states to localStorage so a wall tablet survives reloads.

07  Data & device boundary

Build UI-first against a mock store, but define the seams now so real integration is a drop-in.

Mock now (recommended)
  • Seed room + media + calendar from a static fixtures file.
  • Controls mutate the local store optimistically.
  • Climate stats (temp/humidity/energy/outdoor) are read-only display for now.
Real later (seam)
  • Wrap device I/O in a HomeAdapter interface: setBrightness, setWarmth, setVolume, toggleLight, subscribe().
  • Ship a MockAdapter + leave a MatterAdapter / Home Assistant stub.

08  Build notes & guardrails

  • Stack: Vite + React + TypeScript, plain CSS Modules or vanilla-extract. No Tailwind, no component library — the visual language is specific and light. One theme file for tokens.
  • Touch, not mouse: all drags via Pointer Events with window-level move/up listeners; touch-action:none on dial + slider. Min hit target 44px (chips, transport buttons, swatches already exceed this).
  • Fixed viewport: design is exactly 1280×800; do not make it responsive/fluid for other sizes in v1. Lock the root to the device size; only card-internal regions scroll. Kiosk fullscreen, no browser chrome.
  • No emoji, no gradients as decoration (only the functional dial glow + bottom scroll fade). Cards are flat with 1px borders — keep it that way.
  • Accessibility: every icon-only button gets an aria-label; the dial/slider need role="slider" + keyboard arrow support (prototype omits this — add it).
  • Don't ship the DC runtime. The reference is authored in a prototyping format (<x-dc>, sc-for, renderVals). Read it for values and logic only; re-author as idiomatic React components.
Definition of done
Screen renders at 1280×800 matching the reference; dial + volume drag correctly by touch; room switch swaps all center-card state; light on/off + warmth linkage work; Media/Calendar tabs switch and scroll; accent theme + 12/24h toggle apply everywhere; state persists across reload.
